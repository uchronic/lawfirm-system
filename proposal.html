01

项目背景与核心需求

核心痛点

纸质档案与电子文件分散管理，查阅耗时；相关法条与案例依赖人工检索，效率低下；历史档案归档标准不一，无法有效复用。

系统目标

实现全档案数字化入库与智能分类；自动关联全国现行法律法规；AI辅助生成案件分析报告；所有数据严格本地化，当事人信息不出所内网络。

文档来源

新案件：当事人通过浏览器在线填写结构化建档表单，提交即入库。
历史档案：扫描件（图片/PDF）批量上传，AI自动OCR识别与归类。

部署原则

100% 本地服务器运行，AI推理在内网完成，零云端调用，客户数据物理隔离。符合律师行业对委托人信息保密的职业要求与合规要求。

📝

模块一：在线建档

当事人通过浏览器填写结构化表单（民事/刑事/商事/行政四类），提交后自动生成案件档案、分配编号、生成委托书草稿。无需安装任何软件。

🔍

模块二：历史档案入库

批量上传扫描件（图片/PDF），PaddleOCR自动提取文字，AI识别文档类型（判决书/起诉状/证据材料等），关联案件自动归档。

⚖️

模块三：法条知识库

预加载全国3000余部现行有效法律全文，向量化存储。输入案情描述，语义搜索精准匹配相关法条，支持条文级别定位。

🤖

模块四：AI案件分析

本地大模型读取档案材料，自动提取案件要素、识别争议焦点、推荐适用法条、检索相似历史案件，输出结构化分析报告。

🗂️

模块五：档案管理台

全档案语义搜索、案件状态追踪、批注协作、分析报告管理。三级权限体系：实习生/律师/合伙人，数据访问精细管控。

🔐

模块六：隐私隔离

客户端与律师端网络物理隔离；敏感字段数据库加密存储；AI推理全程在本地完成；文件存储于加密卷。零云端，零数据出境。

03

隐私合规与安全架构

🏠

数据不出境

所有AI推理、检索、分析均在所内服务器完成，无任何云端API调用，当事人信息物理上无法离开内网

🔑

加密存储

档案文件存储于VeraCrypt加密卷；数据库中身份证号、联系方式等敏感字段列级加密

🛡️

访问隔离

客户建档专用子网，提交后即关闭访问；律师系统通过内网IP白名单限制；三级角色权限精细管控

合规说明：本方案满足《律师事务所管理办法》对客户信息保密的要求，以及《个人信息保护法》关于最小化原则和存储安全的规定。AI分析模型完全本地部署，不依赖任何第三方AI服务，委托人信息无任何外泄风险。

软件技术栈

后端框架
Python + FastAPI
高性能异步API，支持文档处理、OCR、AI调用的并发执行
AI 大模型
Ollama + Qwen3.6
阿里巴巴开源中文大模型，本地推理，对中文法律文本理解优秀
OCR 识别
PP-StructureV3 + PaddleOCR-VL
百度开源中文OCR，版式理解+表格+文字框提取；VL模型可选，满足高精度扫描件场景
语义 NER 提取
HaS Text 0.6B + 正则兜底
轻量语义命名实体识别，从文档中自动抽取原告/被告/案号/法院/金额等20+法律字段；正则补全身份证号、手机号等高确信度字段
向量数据库
ChromaDB
本地持久化向量库，实现法条与案件的语义相似度检索
结构化数据库
PostgreSQL
存储档案元数据、当事人信息、案件状态、权限控制
向量嵌入模型
BGE-Large-ZH
北京智源开源中文语义嵌入模型，本地运行，专为中文法律检索优化
PDF 解析

视觉特征识别
LocateAnything-3B + OpenCV

数据隔离
Tenant-per-client 目录隔离

管理界面
HTML + Bootstrap 5
纯浏览器访问，无需安装任何客户端软件，所内任意电脑可用
法条数据源
国家法律法规数据库

实施计划（预计6–7周）

1

第一阶段：基础建设 第1–2周

  • 服务器组装、系统安装、网络隔离配置（VLAN划分）
  • PostgreSQL数据库建模（案件/当事人/档案/权限）
  • FastAPI后端基础框架搭建
  • 客户在线建档表单（民事/刑事/商事/行政四类）
  • 基础档案管理界面（上传/查看/搜索）
2

第二阶段：档案入库 第2–3周

  • PP-StructureV3 集成，批量扫描件处理流水线（OCR + 版式理解）
  • HaS Text 0.6B 语义NER：自动提取原告/被告/案号/法院/金额等法律实体字段
  • AI文档类型自动识别（判决书/起诉状/证据等），LocateAnything检测印章、签字
  • 历史档案批量导入工具（支持拖拽上传）
  • 档案加密存储配置（VeraCrypt加密卷 + per-client目录隔离）
3

第三阶段：法条知识库 第3–4周

  • 下载国家法律法规数据库全文（3000+部）
  • 文本清洗、按条文分段切片
  • BGE-Large-ZH向量化，入库ChromaDB
  • 法条语义搜索接口与界面
4

第四阶段：AI分析引擎 第4–5周

  • Ollama + Qwen3.6模型部署与调优
  • RAG流水线：档案 → 向量检索 → 大模型分析
  • 分析报告模板（要素提取/争议焦点/法条推荐/相似案件）
  • 分析结果持久化与导出（PDF/Word）
5

第五阶段：测试与交付 第5–7周

  • 全流程测试（建档→入库→检索→分析）
  • 权限体系验证（实习生/律师/合伙人三级）
  • 历史档案批量导入（律所提供扫描件）
  • 使用培训（1–2次，3小时/次）
  • 上线后2周内跟进支持

后续扩展能力

对接裁判文书网

可扩展接入中国裁判文书网历史判决数据，将百万量级真实案例纳入相似案件检索库，显著提升案例参考质量。

合同智能审查

对上传的合同文本自动审查条款风险、识别异常条款、对比行业标准合同，生成合同风险报告。

案件进度推送

当事人可通过专属链接查看案件进度，律师设置节点提醒，系统自动推送通知（短信/邮件）。

多模型升级

系统架构预留更强模型接入能力。随硬件升级或新一代中文法律模型发布，可无缝切换更高质量的AI模型。
